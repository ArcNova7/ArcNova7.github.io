常见 CSP 配置要点与可用绕过

什么是 CSP

CSP（Content Security Policy，内容安全策略），是网页应用中常见的一种安全保护机制，通过安全策略的设置来决定外部来源引入的脚本、图片、框架是否能够在当前页面中被调用或执行，CSP 可以通过响应包头或 HTML 中的元标签中的某些字段来实现，CSP 是一种浏览器响应策略。

CSP HTTP 头格式如下：

default-src

default-src 是所有资源类型的默认源。如果没有为某种特定的资源类型指定单独的策略，它会继承 default-src 的设置。

这个策略表示默认情况下只允许加载来自同一域（self）的资源，外部域的资源加载将被阻止。

script-src

script-src 用于控制 JavaScript 脚本的加载源，防止 XSS 攻击。

  • default-src ‘none’、default-src ‘self’.

  • 绕过要点：

    • 不限制“顶级导航”。若未设 navigate-to，仍可整页跳转（location.href、）。

  • script-src ‘nonce-‘：用随机 nonce 放行“受信脚本”（可用于内联与外链脚本标签）。

  • script-src ‘self’ 允许同源外链；’unsafe-inline’ 允许内联；’unsafe-eval’ 放开 eval 类；’strict-dynamic’ 使“受信脚本动态创建的脚本”自动受信，忽略域白名单。

  • 绕过要点：

    • 已存在“带正确 nonce 的脚本标签”会被允许加载外链脚本；如果你能改变它“指向哪里”，即可让浏览器执行你域的 JS。常见方式：
      • 注入 改变文档后续相对 URL 的解析基准，从而让“后续的带 nonce 的相对脚本标签”加载到你的 VPS。
      • 若误配了 data:、blob:，可用 Blob URL/data URL 作为脚本载体。
      • 若出现 ‘unsafe-inline’：直接 、
